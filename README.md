# miniguia-estudos-notebooklm-ons
desafio da DIO
Plano do Projeto: Previsão de Carga Elétrica com IA para o Desafio DIO
1. Contexto e Objetivos
1.1 Contexto
Este projeto visa atender ao desafio da DIO em parceria com o Bradesco, que busca profissionais com perfil de destaque no mercado, especialmente para vagas de análise de carteira de investimento. A proposta é desenvolver um caderno temático no NotebookLM, utilizando a Inteligência Artificial como ferramenta de aprendizagem ativa, e documentar todo o processo em um repositório GitHub.

Aproveitando a formação em Engenharia Elétrica e o interesse em análise de dados, o projeto se concentrará na previsão de carga elétrica com dados reais do Operador Nacional do Sistema Elétrico (ONS). Esta abordagem estratégica busca unir o conhecimento técnico em engenharia com as habilidades analíticas demandadas pelo mercado financeiro, demonstrando uma compreensão aprofundada de um setor crucial para investimentos, como o de energia. A previsão de carga elétrica é um fator determinante para a precificação de energia, avaliação de ativos de geração e transmissão, e gestão de riscos em portfólios de investimento.

1.2 Objetivos de Estudo
Os objetivos de estudo com este material são:

•	Demonstrar proficiência na aplicação de técnicas de Engenharia de IA e Machine Learning para problemas do mundo real, especificamente na previsão de séries temporais.
•	Construir um portfólio técnico robusto, com um projeto de ponta a ponta que abranja desde a exploração de dados brutos até a preparação para deploy de modelos de ML.
•	Desenvolver habilidades em curadoria de fontes e organização do conhecimento, utilizando o NotebookLM como ferramenta central.
•	Aprimorar a engenharia de prompts para extrair informações relevantes e insights de modelos de linguagem, documentando o processo de iteração e resolução de problemas.
•	Criar um miniguia de estudo abrangente sobre previsão de carga elétrica, incluindo resumos estruturados, glossário de termos e prompts reutilizáveis.
•	Conectar o conhecimento técnico de Engenharia Elétrica com as necessidades do mercado financeiro, destacando a relevância da análise de dados de energia para a tomada de decisões de investimento.

3. Engenharia de Prompts e "Cicatrizes"
Esta seção documenta as perguntas estratégicas elaboradas para o NotebookLM, as variações de prompts testadas, as respostas obtidas, as referências utilizadas e as dificuldades encontradas (troubleshooting) para otimizar as respostas da IA. O objetivo é registrar o raciocínio por trás dos resultados e as lições aprendidas no processo de interação com a IA.

3.1 Exemplos de Prompts e "Cicatrizes"
Para ilustrar o processo de engenharia de prompts, apresentamos alguns exemplos baseados no conteúdo do ebook e nas etapas do projeto:

Exemplo 1: Entendimento Inicial e Exploração de Dados
•	Prompt Inicial: "Com base no ebook 'Engenharia de IA na Prática', resuma os principais desafios e as etapas iniciais para explorar os dados de carga elétrica do ONS. Quais são os pontos críticos a serem observados?"
•	Resposta Esperada (Síntese do NotebookLM): O NotebookLM deve identificar que os desafios incluem dados sujos e espalhados, a necessidade de um pipeline de limpeza e a importância de entender as características dos dados (colunas, tipos, período, padrões visíveis) antes de modelar. Pontos críticos seriam a sazonalidade, padrões semanais, feriados e o degrau da MMGD [5].
•	Referência: Ebook: Engenharia de IA na Prática, Seções 1 e 3 [5].
•	Dificuldade/Cicatriz: "O NotebookLM inicialmente focou apenas nos aspectos técnicos do Python, sem contextualizar a importância dos dados do ONS para o setor elétrico. Foi necessário refinar o prompt para incluir 'importância para o setor elétrico' e 'pontos críticos a serem observados' para obter uma resposta mais abrangente e alinhada com o objetivo de negócio."

Exemplo 2: Pipeline de Limpeza de Dados
•	Prompt Inicial: "Explique o conceito de 'pipeline de limpeza de dados' conforme descrito no ebook, e por que a reprodutibilidade é crucial nesta etapa. Quais são as principais funções de um pipeline robusto?"
•	Resposta Esperada (Síntese do NotebookLM): O NotebookLM deve descrever o pipeline como um script que aplica regras de tratamento de dados de forma reprodutível, garantindo que novas entradas sejam processadas consistentemente. A reprodutibilidade é crucial para a profissionalização do projeto. As funções principais incluem carregamento de dados brutos com validação, filtragem e ordenação, tratamento de buracos na série (interpolação) e engenharia de features de calendário [5].
•	Referência: Ebook: Engenharia de IA na Prática, Seção 4 [5].
•	Dificuldade/Cicatriz: "A IA não detalhou suficientemente a analogia industrial da 'estação de tratamento de água' para o pipeline de limpeza. Um prompt de acompanhamento foi: 'Desenvolva a analogia da estação de tratamento de água para o pipeline de limpeza de dados, destacando como cada componente se relaciona.' Isso ajudou a IA a aprofundar a explicação, tornando-a mais didática."

Exemplo 3: Treino de Modelos e MLflow
•	Prompt Inicial: "Quais são os três conceitos fundamentais antes do código para o treino de modelos de Machine Learning em séries temporais, de acordo com o ebook? Como o MLflow se encaixa nesse processo?"
•	Resposta Esperada (Síntese do NotebookLM): O NotebookLM deve abordar a divisão temporal (treino no passado, teste no futuro), a importância de um baseline ingênuo e a criação de lags para dar memória ao modelo. O MLflow é apresentado como uma ferramenta para registrar experimentos, parâmetros e métricas, garantindo a rastreabilidade e comparabilidade dos modelos [5].
•	Referência: Ebook: Engenharia de IA na Prática, Seção 5 [5].
•	Dificuldade/Cicatriz: "A IA inicialmente não enfatizou a importância do shift(1) antes do rolling() na criação de lags para evitar vazamento de dados. Foi necessário um prompt específico: 'Explique a importância do shift(1) na criação de lags para a média móvel, conforme mencionado no ebook, e o risco de vazamento de dados.' Isso gerou uma explicação mais detalhada sobre o tema."

Esses exemplos servem como um guia para você interagir com o NotebookLM. O importante é documentar suas próprias interações, as perguntas que você faz, as respostas que obtém e, principalmente, os ajustes que você faz nos prompts para conseguir a informação desejada. Essas 'cicatrizes' são valiosas e demonstram seu raciocínio crítico e capacidade de solucionar problemas.
